# 物流案件一括一元投稿アプリ (OneLogi-Post)

## 1. 目的と概要
複数の配車マッチング系サービス（Webkit、トラボックス等）に対して、手動での重複入力を排除し、1回のフォーム入力で各プラットフォームへ一括投稿（または選択投稿）を行えるWebアプリケーション。
現場の配車業務の工数を削減し、車輪の再開発を避けるために最新のモダンな自動化ライブラリを採用する。

## 2. 成し遂げたいこと (コア機能)
- **一元入力フォーム**: 積地、卸地、荷物情報、金額、車種、連絡先などを1画面で入力。
- **投稿先動的選択**: 画面上のチェックボックス（横並び配置）で、送信したいプラットフォーム（Webkit / トラボックス等）をオンオフできる。
- **一括自動送信**:
  - **トラボックス**: API非公開のため、モダンなWeb自動化（Playwright）を用いてヘッドレス/有頭ブラウザでログインからフォーム投稿までを自動制御。
  - **Webkit**: 今後提供されるAPIキーを用いたAPIリクエスト（requests）によるデータ送信（将来拡張）。

## 3. 技術選定 (刷新版)
最新のClaude Codeが最も効率的にコード生成・テスト・自動化を行えるスタックに刷新。

- **Backend**: Python 3.11+ / FastAPI (Flaskより非同期処理・堅牢な型安全に優れ、Playwrightとの親和性が高い)
- **Frontend**: HTML5 / Tailwind CSS (CSSの手動管理をなくし、デザイン崩れを防ぐ)
- **Browser Automation**: Playwright for Python (Seleniumに比べ、自動待機が標準、コード生成機能 `codegen` があり、要素変更に強い)
- **Environment Management**: venv / pip

## 4. 画面・レイアウト仕様
- ログイン画面（アプリケーション自体の簡易認証）
- 案件登録画面
  - 各種物流入力項目（日付、時間、都道府県、市区町村、重量、車格、運賃等）
  - 「投稿先」セクション（チェックボックスと文字が横並び、かつ各グループ同士も横並びで配置）
  - 「案件を登録」ボタン
